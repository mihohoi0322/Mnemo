# 要件定義（ドラフト）

## 1. 目的 / ゴール
- スマホで撮ったスクリーンショットをAIで自動タグ付けし、後から自然文で検索できるようにする。
- 個人のメモ整理が主用途（例: 本・記事・UI・レシピ・買い物候補などを探しやすく）。

## 2. 想定ユーザー
- 個人ユーザー（情報収集・備忘・整理が目的）。
- テック/非テック問わず、スクリーンショットをよく使う層。

## 3. コア体験（ユーザーストーリー）
- ユーザーはスクリーンショットを保存するだけで、AIが自動で内容を理解しタグを付与してくれる。
- ユーザーは自然文で検索できる（意味ベース検索を優先）。
  - 例: 「青い表紙の本」「料理のレシピが写ってるやつ」
- ユーザーはタグを編集・追加・削除できる。

## 4. 機能要件（MVP）
- 画像の取り込み
  - iOSアプリでスクリーンショットを保存・収集。
  - ローカル保存が前提。
  - 端末権限は初回の取り込み時に要求（目的と使い方の説明を表示）。
- AI解析
  - 画像内容の解析（OCR＋物体/シーン推定）。
  - 自動タグ付け（タグは編集可能）。
  - 解析はクラウドで実施してよい。
- 検索
  - 自然文検索（意味検索）。
  - タグ検索（手動タグ含む）。
- 整理
  - コレクション（任意の箱）を作成し、画像を追加できる（例: 本/買い物/レシピ）。
  - ライブラリ（一覧）から複数選択して、タグ/コレクションの一括操作ができる。
- 編集
  - タグの編集（追加/削除/修正）。
  - タグの信頼度表示（任意）。
  - 画像の削除（完全削除、復元なし）。

## 5. 非機能要件
- プライバシー
  - 画像の保存はローカル。
  - 解析はクラウド利用可。
- パフォーマンス
  - 検索レスポンス: 体感で即時（目安1秒以内）。
- セキュリティ
  - 端末ローカルの暗号化（要検討）。

## 6. 将来拡張（vNext）
- Azure連携（同期/検索/AI処理）。
- Webアプリ（閲覧・検索）。
- ユーザー間の共有（要検討）。

## 7. 対応プラットフォーム
- iOS（優先）。
- Web（可能なら）。

## 8. 未確定 / 要確認
- 想定枚数・データサイズ（スケール要件）。
- 検索品質の評価基準（満足度・再現率など）。
- オフライン時の振る舞い（解析待ちキューなど）。
