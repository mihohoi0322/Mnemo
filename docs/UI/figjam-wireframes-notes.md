# FigJamワイヤー（MVP）メモ

対象FigJam（ワイヤー）:
- Mnemo Wireframes (FigJam / MVP)

## 目的
- 検索起点の体験を崩さず、後から整理（タグ/コレクション/一括操作）できる画面構成を確定する。

## 画面ごとの決めごと（MVP）

### 検索（ホーム）
- 検索バーは常時表示。
- 下部は「最近見た」「最近取り込んだ」「解析待ち/失敗」を並べ、探し始めの導線にする。

### 検索結果
- フィルタは3つまでに絞る（タグ/コレクション/期間）。
- 整理は検索結果でも一括操作できる（棚卸しに行かなくても良い）。

### 詳細
- 整理の最短導線は「タグ編集」と「コレクションに追加」の2つ。
- 自動タグと手動タグは見た目で区別する。
- 削除は完全削除（取り消し不可）。確認ダイアログで明示する。

### ライブラリ
- 一括操作の主戦場。
- 解析状態でのフィルタ（待ち/失敗）を必ず置く。

### コレクション
- MVPは単一階層（ネストなし）。
- 作成/名称変更/削除 + 中身一覧。

### 解析キュー
- 失敗の再試行ができる。
- 失敗理由は短文で良い（ユーザーが次の行動を取れることが重要）。

### 設定
- クラウド解析ON/OFF。
- Wi-Fiのみ等の送信条件。

## 未決定（次の論点）

- コレクションの作成導線（ホームから作成 vs 追加時に作成）。
