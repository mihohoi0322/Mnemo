# UI要件（MVP / 検索起点 + 整理）

## 1. IA（情報設計）
- 画像（スクリーンショット）: 1枚ごとの基本単位。
- タグ: AIが自動付与するタグと、人間が付与する手動タグ。
- コレクション: 人間の意図でまとめる箱（例: 本/買い物/レシピ）。

## 2. ナビゲーション
- タブ（推奨）
  - 検索（主導線）
  - ライブラリ（棚卸し/整理・一括操作）
  - コレクション（箱の管理・閲覧）
  - 設定

## 3. 画面一覧（必須）

### A. 検索（ホーム）
- 目的: 自然文で「思い出して見つける」を最短で。
- 主要要素
  - 検索バー（常時表示）。
  - サジェスト（最近の検索/よく使うタグ/コレクション）。
  - セクション
    - 最近見た
    - 最近取り込んだ
    - 解析待ち/失敗（ステータスへの導線）
- 主要挙動
  - 入力中に結果プレビュー（軽量）。
  - 検索確定で結果一覧へ。
  - 初回はクラウド解析の同意モーダルを表示（ON/OFFと送信内容の説明）。
  - 初回の取り込み操作で端末権限を要求（権限の目的説明を表示）。

### B. 検索結果
- 目的: 「候補の中から当たりを選ぶ」体験。
- 主要要素
  - 結果グリッド（サムネイル）。
  - 絞り込み（タグ、コレクション、期間）。
  - 並び替え（関連度/新しい順）。
  - 複数選択（整理アクションへ）。

### C. 詳細（画像 + 情報）
- 目的: 画像の内容確認と、最小の手間で整理。
- 主要要素
  - 画像プレビュー（ズーム）。
  - OCRテキスト（折りたたみ可）。
  - タグ（自動/手動の区別が分かる表示）。
  - コレクション追加（ワンタップ）。
  - 関連候補（任意）。
  - 削除（完全削除、確認ダイアログ必須）。

### D. タグ編集
- 目的: AIタグを信頼しつつ、ユーザーが最終決定できる。
- 主要要素
  - タグ追加（入力 + サジェスト）。
  - タグ削除。
  - 自動タグのオン/オフ（非表示ではなく除外扱いの挙動は要検討）。

### E. ライブラリ（一覧）
- 目的: 探す以外の「整理」をまとめて行う。
- 主要要素
  - 全件グリッド。
  - フィルタ（解析状態、タグ、コレクション）。
  - 複数選択。
- 一括アクション
  - タグ一括付与/削除。
  - コレクション一括追加。
  - 削除（完全削除、確認ダイアログ必須）。

### F. コレクション
- 目的: 人間の箱（本棚）でまとまって見返す。
- 主要要素
  - コレクション一覧（作成/名称変更/削除）。
  - コレクション詳細（中の画像一覧、並び替え）。

### G. 解析キュー（進捗/失敗）
- 目的: 「解析がまだ」でも不安にさせない。
- 主要要素
  - 状態: 解析待ち/解析中/成功/失敗。
  - 失敗理由（簡易）と再試行。

### H. 設定
- 目的: プライバシーと挙動のコントロール。
- 最低限
  - クラウド解析のON/OFF（ON時の送信内容の説明）。
  - モバイル通信時の解析（Wi-Fiのみ 等）。
  - 初回同意の再確認（同意文と送信内容の説明）。

## 4. 状態設計（最低限）
- 解析前: OCR/タグなし（プレースホルダ表示）。
- 解析中: 進捗インジケータ。
- 解析失敗: 再試行導線 + ローカル保存は継続。
- タグゼロ: 「タグを追加」導線。
- 結果ゼロ: クエリ改善ヒント（例: もう少し具体的に）。
- 削除: 取り消し不可（確認ダイアログで明示）。

## 5. UIの原則（MVP）
- 検索バーは最短で触れる位置に固定（検索起点）。
- 整理は「詳細で1タップ」「一覧で一括」の二段構え。
- 自動タグと手動タグは見た目と意味が混ざらない。
