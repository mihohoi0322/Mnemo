# UIラフフレーム指示書（全画面一気に並べる用）

## 0. 前提
- 画面サイズ: iPhone 14 Pro（393 x 852）
- 余白: 左右 16 / セクション上下 16〜24
- グリッド: 8pt

## 1. 共通テンプレ（各画面の骨格）
1. ヘッダー
   - 高さ: 56
   - 左: 画面タイトル（24/32）
   - 右: アイコン or ボタン
2. 検索バー（必要画面のみ）
   - 高さ: 44
3. セクションヘッダー
   - 高さ: 24
   - 右に "See all" を置いても良い
4. カード/グリッド枠
   - サムネ: 160 x 160（2列）
   - カード間の余白: 12
5. 下部アクションバー（必要画面のみ）
   - 高さ: 56

## 2. 画面ごとのラフ構成

### A. 検索（ホーム）
- ヘッダー
- 検索バー
- サジェスト（横スクロールチップ）
- 最近見た（横スクロールサムネ）
- 最近取り込んだ（横スクロールサムネ）
- 解析待ち/失敗バナー（小さめ）

### B. 検索結果
- 検索バー（戻る付き）
- フィルタ行（タグ/コレクション/期間）
- 結果グリッド（2列）
- 下部アクションバー（複数選択時）

### C. 詳細
- 画像ビュー（全幅、縦 280〜320）
- クイック操作（タグ編集 / コレクション追加 / 削除）
- タグ（自動/手動を区別）
- OCRテキスト（折りたたみ）
- 関連候補（横スクロール）

### D. タグ編集（モーダル）
- タイトル
- タグ追加入力
- サジェスト
- 現在タグ一覧
- 保存 / キャンセル

### E. ライブラリ
- ヘッダー
- フィルタ（解析状態/タグ/コレクション）
- 全件グリッド
- 下部アクションバー（複数選択時）

### F. コレクション
- ヘッダー + 作成ボタン
- 一覧（カード or リスト）
- コレクション詳細: 中身グリッド

### G. 解析キュー
- ヘッダー
- 状態別セクション（待ち/中/失敗）
- 失敗は再試行CTA

### H. 設定
- ヘッダー
- クラウド解析 ON/OFF
- 通信条件（Wi-Fiのみ等）
- 将来項目（バックアップ/エクスポート）

## 3. 画面配置のコツ
- 左から右へ並べる（検索 → 結果 → 詳細 → ライブラリ → コレクション → 解析 → 設定）。
- 画面間に 80〜120 の水平余白を確保。
- 画面上部にラベル（画面名）を付与。
