# 非機能要件（詳細）

## 1. パフォーマンス
- 検索レスポンス: 1秒以内（1,000件程度）
- スクロール時のフレーム落ちなし（60fps目標）
- 解析結果反映: 5秒以内（理想）

## 2. プライバシー
- 画像保存はローカルのみ
- 解析送信時はユーザーが明示的に同意
- 送信対象は「画像本体 + 必須メタデータ」のみ

## 3. オフライン
- オフライン時は解析をキューに保持
- 再接続時に自動再送
 - 自動再試行は最大3回（指数バックオフ: 30秒 / 2分 / 5分）

## 4. セキュリティ
- TLS必須
- 解析サービス側は画像を永続化しない
- ローカルDBは端末暗号化領域を利用（可能なら）

## 5. 可用性
- 解析サービスは99.5%稼働を目標
- 失敗時は再試行可能

## 6. 拡張性
- Azure同期を前提にIDはグローバル一意
- 将来Web版に備えてAPIを拡張可能に設計
